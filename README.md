# NikkiNikki-wardrobe
网页版工具，下载到本地解压点击html文件即可用，专门用于通过视频流（屏幕共享）实时识别并记录《奇迹暖暖》衣柜中的衣服编号。

---

## 🌟 核心亮点

* **部位独立存储**：支持发型、连衣裙、饰品等 10 大部位。切换部位时，已抓取的编号会自动保存在对应分类下，互不干扰。
* **智慧逻辑纠错**：
    * **自动补位**：识别到 `70` 且根据排序逻辑应为三位时，自动修正为 `070`。
    * **首位丢失补偿**：当识别到编号突然变小（如 `998` 后面跟着 `002`），系统会智能判断是否丢失了千位数的 `1`，并自动补偿为 `1002`。
    * **步长过滤 (MAX_STEP)**：如果前后两个编号跳跃过大（默认 > 400），系统会视其为“幻觉误读”并自动剔除。——可修改参数
* **高精度预处理**：内置针对游戏粉色文字颜色的二值化算法，通过高倍率缩放和噪点过滤，极大提升了对复杂背景衣服的识别成功率。
* **本地运行**：无需上传任何图像到服务器，所有识别过程均在你的浏览器本地完成，安全隐私。

## 🚀 如何开始使用

### 第一步：准备环境
1.  建议使用 **电脑端 Chrome 或 Edge 浏览器**（移动端浏览器通常不支持屏幕共享功能）。
2.  在电脑上连接手机，手机屏幕可以在电脑上查看（具体情况每台手机可能不一致，可以自行搜索如何见手机界面投到电脑上）
3.  在手机上打开《奇迹暖暖》模拟器，并进入“衣柜”界面。

### 第二步：启动工具
1.  双击打开 `Nikki_wardrobe.html` 文件。
2.  等待上方状态栏显示 **“✅ 引擎就绪”**（首次加载可能需要几秒钟下载 OCR 语言包）。

### 第三步：开始捕获
1.  在工具页面点击 **“1. 开始捕获”**。
2.  在弹出的系统窗口中，选择你的**模拟器窗口**进行共享。
3.  点击右侧的部位标签（如“发型”、“连衣裙”），确保当前选择与游戏内一致。

### 第四步：扫描与收集
1.  缓慢滚动游戏衣柜界面。
2.  观察右侧的“调试画布”，如果数字清晰且呈黑白对比，识别效果最佳。
3.  识别到的编号会实时以粉色标签形式出现在下方。

## 🧠 智慧识别参数说明

如果你发现识别结果不够准确，可以在代码中微调以下两个核心参数（搜索 `// --- 全局控制参数 ---`）：

| 参数名 | 默认值 | 作用 |
| :--- | :--- | :--- |
| **`MAX_STEP`** | `400` | 允许相邻两个编号的最大差值。防止把背景纹理误读成一个巨大的无关数字。 |
| **`MIN_CONFIDENCE`** | `82` | OCR 置信度门槛。数值越高越严谨，数值越低抓取越快但错字可能增多。 |

---

## ⚠️ 注意事项与技巧

* **不要滚太快**：OCR 识别需要约 0.8 秒的循环时间，滚动过快会导致漏抓。
* **保持窗口比例**：工具是基于特定的屏幕比例进行裁剪的。如果识别不到数字，请尝试调整模拟器窗口的大小或调整代码中的 `cropX/cropY` 比例。
* **饰品特殊处理**：饰品分类允许 3~5 位数字（支持 23999 以前的超长编号），而其他部位限制在 4 位以内。
* **一键复制**：点击“一键复制当前”，可以直接将该部位的所有编号以空格分隔形式存入剪贴板，方便导入 Excel 或其他攻略工具。

## 📋 常见问题 (FAQ)

**Q: 为什么识别出的数字明明只有 400 多，结果里却出现了 1400？**
A: 这是因为触发了“首位 1 补偿逻辑”。如果识别器认为前一个数是 900 多，而当前数被误读成了很小的数，它会尝试加 1000。你可以通过提高 `MIN_CONFIDENCE`（置信度）来减少这种误读。

**Q: 点击“开始捕获”没反应？**
A: 请检查浏览器是否禁用了媒体流权限，或者尝试刷新页面重试。

**Q: 数据会丢失吗？**
A: 只要不刷新页面或关闭标签页，不同部位的数据都会保留。如果你需要永久保存，请务必及时点击“一键复制”保存到本地文档。
